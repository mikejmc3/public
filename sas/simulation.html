Simulating Data in SAS

Let's look at a few ways we can simulate data in SAS. Many approaches of varying complexity exist. For this notebook, we'll stick to fairly straightforward use cases.

Simulating Uncorrelated Variables

Let's say we want to simulate 500 observations. We want to pull random values from a normal distribution with a mean of 100 and a standard deviation of 15. Because we want our results to be reproducible, we will set a seed for the random number generator.

We will simulate our values in a DATA step loop using the RAND function, and we will set the random number generator seed using the STREAMINIT call routine. Just for fun, we will simulate two variables in the loop: one variable that has no restrictions on decimal places, and one variable with values rounded to the nearest integer.

* Parameters for our simulation;
%let sim_n_obs = 500;  * Number of observations to simulate;
%let sim_mean = 100;  * Population mean;
%let sim_sd = 15;  * Population standard deviation;
%let rng_seed = 42;  * Random number generator seed;

* Simulate values;
data work.sim1;
    format
        observation 8.
        sim_float 10.5
        sim_int 8.
    ;
    * Set RNG seed for reproducibility;
    call streaminit(&rng_seed);
    * Simulate values in a loop;
    do observation = 1 to &sim_n_obs;
        * Simulate floating point values;
        sim_float = rand("normal", &sim_mean, &sim_sd);
        * Simulate another set of values and round to nearest integer;
        sim_int = round(rand("normal", &sim_mean, &sim_sd));
        output;
    end;
run;

* Examine the first 10 rows;
proc print data=work.sim1 (obs=10);
run;

* Examine descriptive statistics;
proc means data=work.sim1 nonobs n mean stddev skew kurt maxdec=2;
    var sim_float sim_int;
run;

* Compare distributions;
proc sgplot data=work.sim1;
    * Population distribution;
    * We have to provide a numeric variable here (using OBSERVATION);
    * The MU= and SIGMA= options will determine the shape;
    density observation / 
        type=normal(mu=&sim_mean sigma=&sim_sd) 
        legendlabel="Population"
    ;

    * Simulated float values;
    density sim_float /
        type=kernel
        legendlabel="Sim: Unrounded"
    ;

    * Simulated integers;
    density sim_int /
        type=kernel
        legendlabel="Sim: Rounded"
    ;

    xaxis label="Simulated Value";
run;

Depending on specific details such as the number of simulated observations and the scale of the distribution, rounding simulated values can sometimes lead to noticeably "bumpier" looking distributions.

Simulating Correlated Variables

Let's say we want to simulate two variables that are both normally distributed with given means and standard deviations, but we also want those two variables to have a certain correlation with each other. How can we accomplish this?

Let's examine two scenarios. First, we will simulate based on observed characteristics of two variables in a reference data set, and second, we will make up the characteristics to form the basis of our simulation.

These 1,000 simulated values closely match the original variables in the source table, both in terms of their univariate characteristics as well as their correlation.

There are two noteworthy differences between the original and simulated values. First, the values in the source table were integers, and PROC SIMNORMAL produced unrounded values. Second, the simulation produced some values that are nonsensical within this context, such as a career homerun count of -10.217 (see observation #4 above). The simulation seeks to match the distributional characteristics of the source data, but it has no insight into which particular values "make sense."

We could another step to round these simulated values and set a lower bound of 0 if we wish to maintain a greater level of consistency with the source data.

data work.baseball_sim_rd;
    set work.baseball_sim;
    format
        CrHitsRd 8.
        CrHomeRd 8.
    ;
    * Do not permit rounded values to be less than 0;
    CrHitsRd = max(round(CrHits), 0);
    CrHomeRd = max(round(CrHome), 0);
run;

proc print data=work.baseball_sim_rd (obs=10);
run;

* See descriptives and correlation of simulated variables;
proc corr data=work.baseball_sim_rd;
    var CrHitsRd CrHomeRd;
run;

Simulating From Scratch

Now let's pivot away from simulating values that match the characteristics of variables that appear in an existing reference table, and instead simulate values totally from scratch, just like Grandma used to make!

In this simple example, I will manually create a CORR type data table similar to what PROC CORR produced for two variables that I want to simulate. I want the first variable to have a mean of 112, and I want the second variable to have a mean of 97. I will simulate both to have a standard deviation of 15, and we'll target a correlation of 0.61 for these variables.

A word of warning: mathematically impossible correlation matrixes are fairly easy to create. In general, I would use an approach like this to simulate a small handful of variables at most.

* Descriptives for Variable 1;
%let var1_name = iq_group_1;
%let var1_mean = 112;
%let var1_sd = 15;
%let var1_n = 500;

* Descriptives for Variable 2;
%let var2_name = iq_group_2;
%let var2_mean = 97;
%let var2_sd = 15;
%let var2_n = 500;

* Correlation;
%let var1_var2_corr = 0.61;

* Build CORR type data table;
data work.corr (type=corr);  * The TYPE= data set option is key here;
    length
        _type_ $ 15
        _name_ $ 32
        &var1_name 8
        &var2_name 8
    ;

    _type_ = "MEAN";
    _name_ = " ";
    &var1_name = &var1_mean;
    &var2_name = &var2_mean;
    output;

    _type_ = "STD";
    _name_ = " ";
    &var1_name = &var1_sd;
    &var2_name = &var2_sd;
    output;

    _type_ = "N";
    _name_ = " ";
    &var1_name = &var1_n;
    &var2_name = &var2_n;
    output;

    _type_ = "CORR";
    _name_ = "&var1_name";
    &var1_name = 1;
    &var2_name = "&var1_var2_corr";
    output;

    _type_ = "CORR";
    _name_ = "&var2_name";
    &var1_name = "&var1_var2_corr";
    &var2_name = 1;
    output;
run;

* View the data table we created;
proc print data=work.corr;
run;

* Simulate 1,000 observations;
proc simnormal data=work.corr outsim=work.sim2 numreal=1000 seed=42;
    var &var1_name &var2_name;
run;

* View descriptives and correlation of simulated values;
proc corr data=work.sim2 plots=scatter;
    var &var1_name &var2_name;
run;

Conclusions

In this notebook, I provided an example to simulate a single variable (or, alternatively, a set of uncorrelated variables) using the RAND function in a DATA step, and I provided examples of simulating a pair of correlated variables using PROC SIMNORMAL, which may involve either replicating characteristics of variables that already exist in a reference data table or providing the desired characteristics manually.
